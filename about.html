Creation Process of the Application

This web platform, "ISM School Incidents," was conceived as a comprehensive solution to streamline the reporting and management of facility issues within the school environment. The project bridges the gap between teachers, staff, and the maintenance department through a digital-first approach.

Technology Stack & Architecture

  • Frontend User Interface: Built with modern, semantic HTML5 and vanilla JavaScript. Avoided heavy frameworks to ensure lightning-fast load times and high performance on older devices occasionally found in academic environments.
  • Styling & Aesthetics: Custom CSS variables enabling seamless Light/Dark mode transitions. Advanced CSS animations and HTML5 Canvas API for interactive UI elements such as the mouse-trailing dot matrix.
  • Database & Cloud: Firebase Firestore implementation for real-time NoSQL database synchronization. Stores critical incident data securely in the cloud.
  • Media Storage: Cloudinary API integration for seamless handling and hosting of incident photograph uploads directly from the frontend interface.
  • Localization: Integrated custom JSON-based dynamic multi-language system (EN/ES) without relying on external libraries.

Key Features Built

The platform includes an interactive multi-department form (Primary, Middle, High), urgency categorization, and a persistent cooldown system utilizing browser LocalStorage to prevent spamming from single terminals. Everything is glued together by a responsive design ensuring usability across mobile devices and desktop computers alike.
